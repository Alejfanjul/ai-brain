# Contexto do AI-Brain

Este arquivo orienta como usar o conhecimento e projetos deste repositório.

## Guia de Autores

### Seth Godin
- **Domínio:** Marketing, posicionamento, filosofia de negócio
- **Quando usar:** Decisões de direção, "para onde ir", princípios de marketing
- **Não usar para:** Execução técnica, código, ferramentas

### Derek Sivers
- **Domínio:** Filosofia de vida, simplicidade, decisões fundamentais
- **Quando usar:** Clareza sobre o que importa, como pensar sobre escolhas
- **Não usar para:** Táticas, implementação

### Nate
- **Domínio:** IA aplicada, agentes, desenvolvimento de software moderno
- **Quando usar:** "Como construir isso?", pontes entre ideia e execução com IA
- **Não usar para:** Filosofia pura, marketing tradicional

### Joe Hudson
- **Domínio:** Desenvolvimento pessoal, emoções, mindset
- **Quando usar:** Insegurança, bloqueios, como lidar com sentimentos
- **Não usar para:** Estratégia de negócio, técnico

### Bruno Perini
- **Domínio:** Marca pessoal, finanças pessoais
- **Quando usar:** Posicionamento pessoal, investimentos
- **Não usar para:** Filosofia, técnico

## Como trabalhar

1. Ao iniciar conversa sobre um projeto, ler o README.md do projeto
2. Identificar o estágio atual
3. Consultar autores relevantes conforme o tipo de pergunta
4. Atualizar o README.md do projeto conforme decisões são tomadas
