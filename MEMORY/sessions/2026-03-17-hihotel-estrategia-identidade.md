**Data:** 2026-03-17
**Projeto:** HiHotel (Ale + Matheus)
**Tipo:** Discussão estratégica

## Contexto
Ale e Matheus apresentaram proposta de parceria tecnológica (transição CLT→PJ) ao Diego e Heiko do Duke Beach Hotel. A proposta foi bem recebida. Próximo passo: proposta financeira e técnica detalhada.

## Tópicos discutidos

### 1. Modelo de negócio da HiHotel
- Dois produtos: **Concierge** (face pro hóspede, IP da HiHotel) e **SID** (orquestração interna, pertence ao hotel)
- Concierge = cavalo de Tróia (mais fácil vender, gera receita visível) → abre porta pro SID
- Estrutura de receita por hotel: licença mensal Concierge + fee de setup + contrato evolução SID

### 2. IP e o papel do código na era da IA
- Código é barato de produzir com IA → IP não é sobre código
- O valor real está no conhecimento de domínio: saber COMO hotel funciona, O QUE construir
- Decisão: SID do Duke é do Duke (construído CLT). Concierge é da HiHotel.
- Estratégia: levar o conhecimento, não brigar pelo código

### 3. Visão do SID como 3ª geração de IA (Seth Godin)
- SID não é sistema de registro — é sistema que conecta pessoas
- Informação certa, pessoa certa, hora certa = sincronia operacional
- Efeito de rede interno: melhora quando mais funcionários usam
- Visão: "fazer hotéis funcionarem como um organismo"

### 4. Escalabilidade sem virar consultoria
- Playbook interno (não vendido) acelera cada implementação via IA
- Hotel 1 = anos, hotel 5 = semanas, hotel 20 = dias
- Concierge com base instalada = SaaS vendável (múltiplo de valuation)
- HiHotel não é software puro nem consultoria — é domain experts + IA

### 5. Formação de identidade
- Ale = direção estratégica, Matheus = implementação
- Momento de definir mapa estratégico que guie as ações
- Referências: Nate (Jevons Paradox, domain experts viram builders), Seth Godin (3ª geração IA = conectar pessoas)
