# Session sec-audi

**Date:** 2026-03-18 | **Project:** sistema-os

## Summary

Investigacao de seguranca completa do SID (sistema-os) apos dois hospedes receberem mensagens fraudulentas com dados de reservas. Auditamos codigo, credenciais, logs de login e configuracao do Supabase. Conclusao: sem evidencia de vazamento pelo SID — origem provavel e Booking.com, Omnibees ou HITS PMS. Corrigimos vulnerabilidades encontradas e geramos relatorio formal.

## What Was Done

- Auditoria completa do codigo-fonte: credenciais, endpoints, CORS, headers, autenticacao
- Verificacao de que `.env` nunca foi commitado no git
- Confirmacao de que RLS esta habilitado em todas as 63 tabelas do Supabase
- Analise de 200+ registros de login_history — nenhum IP externo ou atividade suspeita
- Correcao: adicionada autenticacao JWT nos endpoints HITS (`/request-report`, `/health`)
- Correcao: adicionada validacao JWT no WebSocket (`/ws/notificacoes`) via query param
- Removido `download_dir` da resposta do health check HITS
- Frontend atualizado para enviar token JWT na conexao WebSocket
- Cherry-pick do fix de seguranca para branch `main` (sem arrastar outros commits)
- Geracao de novo JWT Secret seguro para o `.env` local (producao ja tinha secret forte)
- Relatorio completo gerado em MD e HTML (docs/reports/)

## Decisions Made

- Vazamento NAO veio do SID — sistemas terceirizados sao a origem provavel
- Cherry-pick usado para levar so o fix de seguranca para main, sem os ~50 commits pendentes de development
- JWT Secret do Dokploy (producao) mantido — ja era forte e aleatorio
- Nao implementamos rate limiting especifico em endpoints de hospedes (melhoria futura)
- Supabase Free tem logs de apenas 1 dia — recomendado avaliar upgrade para Pro

## Learnings

- Reverse proxy (Traefik/Dokploy) nao propaga IP real do cliente — todos os logs mostram IPs internos 10.x.x.x. Precisa configurar X-Forwarded-For
- Tabela `data_changes` existe no banco mas nao esta sendo populada — oportunidade de auditoria
- PowerShell usa `curl` como alias para `Invoke-WebRequest` — usar `curl.exe` para o curl real
- Alejandro nao tem familiaridade com curl — explicar comandos de forma didatica
