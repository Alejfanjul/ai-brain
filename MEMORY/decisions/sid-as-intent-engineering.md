# SID como Intent Engineering para Hotéis

**Data:** 2026-02-25
**Origem:** Conexão entre áudios pessoais (Voz 007-010) + newsletter do Nate (2026-02-24)

## Insight Central

O SID (sistema-os) não é apenas um sistema de ordens de serviço. É uma **camada de intent engineering aplicada a hotéis** — o que o Nate descreve como "the most important unsolved problem in enterprise AI right now".

## O Problema (Nate / Klarna)

- Klarna automatizou atendimento com IA: tickets resolvidos em 2 min (antes 11), $60M economizados
- Resultado: destruiu relacionamento com clientes. CEO admitiu o erro na Bloomberg
- A IA otimizou para o objetivo mensurável (velocidade) e ignorou o intent real (retenção, relacionamento)
- **3 eras da IA:** prompt engineering (o que dizer) → context engineering (o que saber) → intent engineering (o que querer)
- Quase ninguém está construindo para intent engineering ainda

## A Conexão com SID

O que Ale descreveu nos áudios é intent engineering para hotéis:

1. **Entrevistar o dono** — capturar o que ele realmente quer do negócio (não o que ele diz, mas o que ele quer de verdade)
2. **SID como camada de inteligência** — sobre a infraestrutura existente (PMSs), adicionar intent
3. **Duke como showcase** — o Klarna que deu certo, porque o intent layer está sendo construído ANTES da automação
4. **Sistema + consultoria** — o modelo de negócio é exatamente intent engineering: extrair, estruturar e operacionalizar os objetivos do hotel

## Por que é diferente

- PMSs existentes (Cloudbeds, Totvs, etc.) = ferramentas operacionais. Zero intent.
- Nenhum sistema hoteleiro codifica "o que esse hotel quer ser quando crescer"
- Hotéis são negócios onde intent importa demais — a diferença entre "resolver pedido do hóspede rápido" vs "criar experiência memorável que gera retorno"

## Frameworks que já existem no ai-brain

- **TELOS** = intent engineering pessoal (já funciona para Ale)
- **PAI** = infraestrutura de conhecimento pessoal
- Ambos podem ser adaptados para o contexto organizacional/hoteleiro

## Modelo de Expansão

```
Duke (showcase real, intent layer completo)
  → Vender para outros hotéis: SID + consultoria de implementação
  → Clientes: hotéis, Airbnbs, pousadas, hospedagens
  → Parceria: Ale (produto/IA) + Matheus (50/50 a definir)
```
